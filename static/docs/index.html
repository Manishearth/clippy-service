The Clippy Service Documentation

This document gives an overview of the various features and parts of the clippy service documentation. Especially, we want to give you easy access to some of the more tricky features this service provides and how you could solve them.

File Structure

The project is split into multiple files, all having full code documentation. The following files are in the repository:

  • main.rs: the entry point for this - and any – rust app, defining the modules, importing the handlers and starting the Iron server.
  • handlers.rs: the handlers to be served
  • github.rs: the github specific backend code, fetching github SHA and redirecting properly
  • clippy.rs: the clippy subprocess runner and output parser
  • helpers.rs: general purpose helper functions used at various places – like redis setup.

Features

If you are interested in one specific feature this service provides and how we solved this, take a look at this list with pointers to the documentation to look at

  • Redis Support
  • Temporary Folders
  • (sandboxed) Sub Processes with output parsing
  • HTTP Requests to Github with JSON Parsing
  • Inline Unicode Emojis
  • In Memory Unzipping of Downloaded Content

Infrastructure / Deploying

As clippy requires features only available in rust nightly, we have to deploy with that nightly version. To build and run the project we have a Docker Setup, running on a Dokku instance, which is automatically deployed using travis.

As we have the same requirement in Development, we provide a Rust-Nightly Vagrant setup, which already preinstalls the neede Redis-Server.
